================================================================================
                  Руководство пользователя: Competency Gap Analyzer v2.0
================================================================================

Данный проект позволяет собирать вакансии с сайта hh.ru, извлекать
востребованные навыки, сопоставлять их с учебными компетенциями и выполнять
gap-анализ для профилей студентов. Включает ML-модель ранжирования навыков
(XGBoost LTR), систему персональных рекомендаций с объяснениями, кластеризацию
вакансий и анализ трендов.

Быстрый старт
================================================================================

Примеры команд:

  # Полный цикл с нуля: сбор IT-вакансий, обучение модели, анализ
  python main.py --it-sector --regions 1,2 --excel
  python main.py --train-model
  python main.py --skip-collection --run-gap-analysis

  # Альтернативный запуск (если вакансии уже собраны)
  python main.py --skip-collection --run-gap-analysis

  # Быстрый тест на малом объёме
  python main.py --query "Python Developer" --max-pages 1 --show-vacancies


Основные параметры
================================================================================

  --query, -q         Поисковый запрос (название профессии, технологии, навыки).
                      Можно писать на русском или английском, например:
                      "Data Scientist", "Python разработчик", "машинное обучение",
                      "SQL", "Java", "аналитик данных".
                      По умолчанию: "Backend developer"

  --area-id, -a       ID региона для поиска.
                      По умолчанию: 1 (Москва)

  --max-pages, -p     Максимальное количество страниц (до 20). Одна страница = 100 вакансий.
                      По умолчанию: 5

  --period, -d        Период поиска в днях (1–30). Вакансии, опубликованные не ранее этого срока.
                      По умолчанию: 30

  --show-vacancies, -v Показать список найденных вакансий в консоли.
                      По умолчанию: выключено

  --skip-details, -s  Пропустить загрузку детальной информации (только базовый поиск).
                      Ускоряет сбор, но навыки будут извлекаться только из кратких полей.
                      По умолчанию: выключено

  --excel, -e         Сохранить результаты в Excel.
                      По умолчанию: выключено

  --no-filter, -nf    Отключить фильтрацию навыков по белому списку (it_skills.json).
                      По умолчанию: выключено

  --train-model       Обучить ML-модель ранжирования навыков на существующих данных
                      и выйти. Рекомендуется выполнять после каждого сбора вакансий.

  --use-llm           Использовать YandexGPT для генерации объяснений к рекомендациям.
                      По умолчанию: выключено

  --skip-collection   Использовать существующие файлы вакансий, не выполняя новый сбор.

  --run-gap-analysis  Запустить gap-анализ после обработки вакансий.
                      По умолчанию: выключено


Расширенные возможности
================================================================================

1. Поиск по всему IT-сектору
   Используйте флаг --it-sector. Скрипт автоматически выполнит поиск по
   предопределённому списку IT-позиций (40+ профессий) с ограничением 500 вакансий
   на каждую позицию и фильтром по профессиональной области industry=7 (IT).
   Можно комбинировать с указанием регионов.

     python main.py --it-sector --regions 1,2 --excel

2. Несколько запросов из файла
   Создайте текстовый файл (например, queries.txt), где каждая строка — отдельный
   запрос. Затем укажите его через --queries-file.

     python main.py --queries-file queries.txt --regions 1,2,88 --excel

3. Несколько регионов
   Укажите через --regions список ID через запятую. Пример: 1,2,88,66
   Полный список популярных регионов приведён ниже.

     python main.py --query "Data Scientist" --regions 1,2,88 --excel

4. Фильтрация по профессиональной области
   Используйте --industry с кодом отрасли. Код 7 соответствует IT. Можно
   комбинировать с любым запросом.

     python main.py --query "Analyst" --industry 7 --excel

5. Ограничение количества вакансий на запрос
   По умолчанию для обычных запросов лимит — 1000 вакансий, для IT-сектора — 500.
   При необходимости можно изменить через --max-vacancies-per-query.

     python main.py --it-sector --max-vacancies-per-query 300 --excel

6. Интерактивный режим
   Запустите без аргументов или с --interactive. Вы сможете выбрать профессию,
   регионы и настройки в пошаговом меню.

     python main.py --interactive


Обучение кластеров вакансий
================================================================================

После сбора вакансий рекомендуется обучить кластеры для улучшения качества
рекомендаций. Доступные уровни: junior, middle, senior, all (по умолчанию).

  python scripts/train_clusters.py --level all

Полная пересборка проекта (после изменений в коде)
================================================================================

Скрипт автоматически удаляет устаревшие кэши и выполняет все шаги:
  python scripts/full_rebuild.py

Что делает скрипт:
  - Удаляет parsed_skills.pkl, модели кластеров, LTR-модель, кэш эмбеддингов
  - Пересчитывает навыки из вакансий
  - Обучает кластеры
  - Обучает LTR-модель
  - Запускает gap-анализ и генерирует рекомендации


Автоматическое расширение белого списка навыков
================================================================================

Для поддержания актуальности it_skills.json используйте скрипт extend_it_skills.py:

  # Интерактивный режим (подтверждение каждого навыка)
  python scripts/extend_it_skills.py

  # Автоматически добавить все новые навыки
  python scripts/extend_it_skills.py --yes

  # Указать конкретный файл вакансий
  python scripts/extend_it_skills.py --vacancies data/result/hh_vacancies_detailed.json

После расширения обязательно очистите кэш:

  # Windows PowerShell
  Remove-Item data/processed/parsed_skills.pkl -ErrorAction SilentlyContinue
  Remove-Item -Recurse data/embeddings/cache/ -ErrorAction SilentlyContinue

  # Linux/macOS
  rm -f data/processed/parsed_skills.pkl
  rm -rf data/embeddings/cache/


Анализ трендов навыков
================================================================================

Позволяет отслеживать динамику спроса на навыки по историческим снимкам.

  # Анализ по всем историческим снимкам (без сохранения нового)
  python -m src.analyzers.trends --no-save

  # По последним 3 снимкам
  python -m src.analyzers.trends --history 3 --no-save

  # Сравнение двух конкретных файлов
  python -m src.analyzers.trends --current data/history/freq_2026-04-26.json --previous data/history/freq_2026-04-15.json --no-save

  # Увеличить чувствительность и количество навыков
  python -m src.analyzers.trends --no-save --min-change 2.0 --top 15

Параметры анализа трендов:

  --current, -c       Путь к файлу текущего снимка
  --previous, -p      Путь к файлу предыдущего снимка
  --history, -n       Количество последних снимков для анализа (по умолчанию все)
  --no-save           Не сохранять новый снимок
  --top               Количество выводимых навыков (по умолчанию 10)
  --min-change        Минимальный процент изменения для тренда (по умолчанию 3.0)

Графики сохраняются в data/result/trends/:
  - trending_skills.png – растущие и падающие навыки
  - skill_timeline.png – временные ряды для топ-10 навыков

Метрики и их интерпретация
================================================================================

После gap-анализа для каждого профиля выводятся следующие метрики:

  Match score (v2)           Общее покрытие рынка (~9%). Отражает потребность
                             в дообучении, а не процент освоенных навыков.
                             Рассчитывается как 70% навыкового + 30% доменного покрытия.

  Confidence                 Готовность к целевому уровню (~7.95%).

  Навыковое покрытие         Средний score дефицитных навыков.

  Доменное покрытие          Среднее покрытие по доменам.

  Реальное покрытие рынка    Доля навыков студента от всех рыночных (~44%).
                             Показывает, сколько из топ-рынка студент уже знает.

  Покрытие рынка (старое)    Для обратной совместимости.


Интерпретация рекомендаций
================================================================================

Каждая персональная рекомендация содержит:

  - Важность (importance_score) – числовая оценка необходимости навыка.
  - Приоритет – HIGH (>0.7), MEDIUM (0.4-0.7), LOW (<0.4).
  - Почему важно – объяснение на основе рыночного спроса и кластерной релевантности.
  - Как учить – рекомендуемый путь обучения с учётом уровня.
  - Время освоения – примерная оценка времени на изучение навыка.

ML-модель и рекомендации
================================================================================

Проект включает современную систему ранжирования навыков (Learning-to-Rank) на
основе XGBoost. Модель предсказывает важность недостающих навыков в процентах
(0–100%) и даёт понятные объяснения (почему навык важен: частота, рыночный вес,
уровень и т.д.).

Порядок использования:

1. Собрать вакансии:
   python main.py --it-sector

2. Обучить модель на собранных данных:
   python main.py --train-model

3. Запустить gap-анализ с рекомендациями (модель загрузится автоматически):
   python main.py --skip-collection --run-gap-analysis

Рекомендации сохраняются в data/result/<профиль>/full_recommendations_<профиль>.json.
Модель можно переобучать после каждого обновления вакансий для актуализации весов.

================================================================================

Часто задаваемые вопросы
================================================================================

Вопрос: Можно ли искать вакансии за пределами России?
Ответ: Да, для этого нужно узнать ID интересующего региона (например, 40 – Казахстан)
       и передать его в --area-id или --regions.

Вопрос: Почему в результате мало вакансий?
Ответ: Проверьте параметры поиска: возможно, запрос слишком узкий, период слишком
       маленький или регион не популярен. Используйте --it-sector для широкого
       охвата.

Вопрос: Как добавить свои регионы в интерактивный режим?
Ответ: В интерактивном режиме при выборе регионов есть возможность ввести
       произвольные ID. Также можно указать их через --regions.

Вопрос: Что делать, если навыки не сопоставляются с компетенциями?
Ответ: Убедитесь, что в файле data/processed/competency_mapping.json
       содержатся корректные соответствия. При необходимости дополните его.

Вопрос: Как часто нужно переобучать ML-модель?
Ответ: Рекомендуется после каждого значительного обновления данных (новый сбор
       вакансий). Это обеспечит актуальность весов и точность рекомендаций.

Вопрос: Почему покрытие рынка такое низкое (~9%)?
Ответ: "Общее покрытие рынка (v2)" — это метрика потребности в дообучении,
       а не процент освоенных навыков. Реальное покрытие рынка показывает,
       сколько из топ-рынка студент уже знает.

Вопрос: Почему в рекомендациях нет Python/Docker, хотя они востребованы?
Ответ: Система исключает навыки, которые уже есть у студента. Если Python или
       Docker присутствуют в профиле, они не будут рекомендоваться.

Вопрос: Почему в логах появляются предупреждения про HF_HUB?
Ответ: Это связано с загрузкой моделей эмбеддингов с Hugging Face. При локальной
       работе можно игнорировать. Для повышения лимитов скачивания получите
       бесплатный токен на huggingface.co и добавьте в .env как HF_TOKEN.

Вопрос: Как проверить корректность системы?
Ответ: Создайте профиль "идеального студента" со всеми кодами компетенций из
       маппинга. Его реальное покрытие должно быть >95%, а рекомендации пустыми.
