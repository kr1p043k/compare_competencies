## CRITICAL (8)

### C1 — joblib.load() RCE
`cache_manager.py:37`, `ltr_recommendation_engine.py:657`, `vacancy_clustering.py:307,326`, `embedding_comparator.py:91`
Пять сайтов десериализуют через `joblib.load()` без проверки целостности. Поддельный `.joblib` файл в `data/cache/` даёт RCE.
**Fix:** HMAC-подпись перед dump, верификация перед load. Замена на `safetensors` для моделей.

### C2 — Docker: контейнер от root
`Dockerfile` — нет `USER approot`. Любой RCE внутри контейнера = root на хосте.
**Fix:** `RUN useradd -m app && USER app` после системных зависимостей.

### C3 — PostgreSQL: дефолт creds + публичный порт
`docker-compose.yml:189,192` — `postgres:postgres` + `5432:5432` на все интерфейсы.
**Fix:** Убрать host mapping,.require не-дефолтный пароль.

### C4 — Ollama без аутентификации на всех интерфейсах
`docker-compose.yml:86` — порт 11434 без auth.
**Fix:** Привязать к 127.0.0.1.

### C5 — Backend API: DEBUG=True + публичный порт
`docker-compose.yml:8`, `.env:46` — стек-трейсы утекают.
**Fix:** `DEBUG=False`, привязка к localhost.

### C6 — API ключи в .env в git-истории
`.env:6-11` — `HH_CLIENT_SECRET`, HuggingFace токен, n8n JWT.
**Fix:** Ротация всех токенов немедленно. Добавить `.env` в `.gitignore` (уже есть, но проверить историю).

### C7 — Пароли в SQL seeds
`sql/001_create_krm_schema.sql:569-580` — `admin`, `teacher123`, `student` в открытом виде.
**Fix:** Генерировать через `gen_salt('bf')` в seed-скрипте.

### C8 — HF mirror hardcoded — supply chain
`Dockerfile:49`, `docker-compose.yml:39` — `hf-mirror.com` как единственный источник моделей.
**Fix:** Параметризовать через ARG/ENV без дефолта.
