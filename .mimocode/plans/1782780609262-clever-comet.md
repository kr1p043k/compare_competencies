## Исправления по приоритетам

### БЛОК 1: CRITICAL Runtime Crashes (6 багов)

**B1.1 — gap_metrics.py: NameError + AttributeError (C3)**
- Файл: `src/monitoring/gap_metrics.py`
- Проблема: `record_analysis()` вызывает undefined `gap_analysis_duration` (должен быть `gap_duration`), и `.observe()` на Counter (должен быть `.inc()`)
- Фикс: `gap_analysis_duration` → `gap_duration`, `.observe(val)` → `.inc(val)`

**B1.2 — krm_models.py: datetime.now() при импорте (C1)**
- Файл: `src/models/krm_models.py:44,64`
- Проблема: `default=datetime.now(timezone.utc)` вычисляется один раз при импорте модуля
- Фикс: `default_factory=lambda: datetime.now(timezone.utc)`

**B1.3 — fix_rpd_data.py: SQL injection (C4)**
- Файл: `src/cli/fix_rpd_data.py:162`
- Проблема: `v['text']` в f-string — SQL injection
- Фикс: Использовать параметризованные запросы через `text().bindparams()` или escape

**B1.4 — runner.py: asyncio.run() + get_pool() = None (C11)**
- Файл: `src/pipeline/runner.py:288-296`
- Проблема: `get_pool()` возвращает None если пул не инициализирован
- Фикс: Проверка `if pool is None:` перед использованием, или инициализация

**B1.5 — db_writer.py: ValueError на нечисловом id (M7)**
- Файл: `src/pipeline/db_writer.py:157`
- Проблема: `int(v.get("id", 0))` без try/except
- Фикс: `try: hh_id = int(v.get("id", 0)) except (ValueError, TypeError): continue`

**B1.6 — seed_users.py: hardcoded creds (C5)**
- Файл: `seed_users.py:10`
- Проблема: `postgres:700009` в коде
- Фикс: Читать из env: `os.environ.get("DATABASE_URL", "postgresql://...")`
