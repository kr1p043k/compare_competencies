## Fix Plan

### Fix 1: `src/predictors/recommendation_engine.py:153-160` — Type mismatch in GapAnalyzer

**Problem**: `fit()` passes `skill_weights: dict[str, float]` to `GapAnalyzer()` which expects `dict[str, dict[str, float]]`.
**Fix**: Wrap in a synthetic level structure or use a dedicated method. Since the recommendation engine doesn't need per-level granularity, wrap as `{"all": skill_weights}`:
```python
self.gap_analyzer = GapAnalyzer({"all": skill_weights})
```

### Fix 2: `src/config.py:133-136` — Readiness weights (redesigned)

**Problem**: Sum is 0.65, `READINESS_DOMAIN_WEIGHT = -0.15` (negative = sign error), and the formula mixes positive/negative semantics incorrectly.
**Fix**: Redesign weights to be logically consistent:

```python
READINESS_MARKET_WEIGHT     = 0.45   # market_coverage_score (already [0,1])
READINESS_SKILL_WEIGHT      = 0.30   # strong_count / total (normalized to [0,1])
READINESS_GAP_PENALTY_WEIGHT = -0.25  # avg_gap (penalty, subtracted)
```

Remove `READINESS_DOMAIN_WEIGHT` entirely — domain coverage is already in `market_coverage_score` via the 0.40 coefficient (line 266: `0.40 * domain_coverage_score`). Keeping it creates double-counting.

With these weights: max readiness = 0.45 + 0.30 + 0 = 0.75 (100% coverage, 100% strong, 0 gap). This is the theoretical ceiling — realistic values will be lower.

Also in the formula: normalize `(strong_count / market_div * 100)` to `(strong_count / market_div)` (drop the `* 100` so it stays [0,1]).

### Fix 3: `src/analyzers/gap/profile_evaluator.py:266-278` — Readiness formula (complete rewrite)

**Problem**: Semantic confusion — DOMAIN_WEIGHT × weak_ratio, GAP_PENALTY × domain_coverage, double-counting of domain_coverage.
**Fix**: Rewrite the formula to match the new weights:
```python
market_div = len(market_skills) if market_skills else 1
strong_count = sum(1 for m in metrics.values() if m.user_level >= 0.7)
avg_gap = sum((m.gap_j + m.gap_m + m.gap_s) / 3 for m in metrics.values()) / max(len(metrics), 1) if metrics else 0.0

readiness = (
    config.READINESS_MARKET_WEIGHT * market_coverage_score
    + config.READINESS_SKILL_WEIGHT * (strong_count / market_div)
    + config.READINESS_GAP_PENALTY_WEIGHT * min(avg_gap, 1.0)
)
readiness = round(max(0.0, min(1.0, readiness)) * 100, 2)
```

### Fix 4: `src/analyzers/gap/profile_evaluator.py:38-53` — Dead parameters

**Problem**: `readiness_weights`, `level_difficulty`, `skill_weights` are accepted but never used.
**Fix**: Wire in `readiness_weights` to allow override of config constants:
```python
# In __init__:
self.readiness_weights = readiness_weights or (
    config.READINESS_MARKET_WEIGHT,
    config.READINESS_SKILL_WEIGHT,
    config.READINESS_GAP_PENALTY_WEIGHT,
)

# In evaluate_profile, replace config.* with:
mkt_w, skill_w, gap_w = self.readiness_weights
readiness = mkt_w * market_coverage_score + skill_w * (strong_count / market_div) + gap_w * min(avg_gap, 1.0)
```

Remove `level_difficulty` and `self.skill_weights` (dead code).

### Fix 5: `src/analyzers/gap/profile_evaluator.py:306` — avg_gap * 100

**Problem**: Gap values from non-normalized weights can exceed 1.0, making the percentage misleading.
**Fix**: In the new formula (Fix 3), cap avg_gap with `min(avg_gap, 1.0)` before multiplying by the gap penalty weight. The `* 100` is removed from avg_gap — the final readiness score is multiplied by 100 at the end. If raw gap values can exceed 1.0 from the GapAnalyzer, add normalization: `gap / max(market_weight, 1.0)` in `GapAnalyzer.compute_metrics()`.

### Fix 6: `src/analyzers/gap/profile_evaluator.py:131-135` — cluster_relevance always 0.0

**Problem**: `0.15 * getattr(metric, "cluster_relevance", 0.0)` always = 0.0.
**Fix**: Use a default fallback value: `0.15 * getattr(metric, "cluster_relevance", 1.0)` or assign a sensible default directly.

### Fix 7: `src/predictors/prophet_forecast.py` — Silent degradation

**Problem**: If Prophet is not installed or fails, system silently falls back to pseudorandom forecasts.
**Fix**: Log a clear warning at startup and in forecast results. Add a `forecast_engine_used` field to output.

### Fix 8: `src/predictors/recommendation_engine.py` — Remove dead GapAnalyzer instance

**Problem**: `self.gap_analyzer` is created but never used.
**Fix**: Remove the dead instance and its creation in `fit()`.
