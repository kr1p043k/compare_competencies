# Улучшение модуля predictions (Prophet + SkillForecast)

## Найденные проблемы

### Критические
1. **skill_forecast.py:153-154 — двойной `datetime.now()`**
   ```python
   predicted = model["intercept"] + model["slope"] * (
       (datetime.now().toordinal() + days_ahead - datetime.now().toordinal())
   )
   ```
   Два вызова `datetime.now()` могут вернуть разные значения. Должно быть просто `days_ahead`.

2. **prophet_forecast.py:175 — мутация результата in-place**
   ```python
   result.ok().engine_used = "genetic_fallback"
   ```
   Мутирует ForecastResult после возврата — fragile паттерн.

3. **skill_forecast.py:67 — `import json` внутри метода**
   Импорт внутри `fit()` вместо уровня модуля.

### Качество кода
4. **prophet_forecast.py:59 — docstring упоминает "GeneticForecastSkill"** — устаревшее название (это SkillForecastEngine).

5. **skill_forecast.py:141 — `days_ahead = 365` хардкод** — ignores `months` parameter, всегда предсказывает на год вперёд.

6. **prophet_forecast.py:148 — `freq="ME"` (Month End)** — может давать непредсказуемые результаты при неравномерных данных.

7. **Нет валидации `months` параметра** — отрицательные или очень большие значения.

8. **skill_forecast.py:159 — асимметричные лимиты роста/падения** — MAX_GROWTH_CAP=3.0 vs -0.5. Неочевидно.

## План исправлений

### 1. Исправить двойной datetime.now() (skill_forecast.py:152-155)

**Было:**
```python
days_ahead = 365
predicted = model["intercept"] + model["slope"] * (
    (datetime.now().toordinal() + days_ahead - datetime.now().toordinal())
)
```

**Стало:**
```python
days_ahead = months * 30  # приближение
predicted = model["intercept"] + model["slope"] * days_ahead
```

### 2. Исправить мутацию in-place (prophet_forecast.py:172-176)

**Было:**
```python
if self._fallback_engine:
    result = self._fallback_engine.forecast(skill, months)
    if result.is_ok():
        result.ok().engine_used = "genetic_fallback"
    return result
```

**Стало:**
```python
if self._fallback_engine:
    result = self._fallback_engine.forecast(skill, months)
    if result.is_ok():
        fr = result.unwrap()
        return Ok(ForecastResult(
            skill=fr.skill,
            current_frequency=fr.current_frequency,
            predicted_growth=fr.predicted_growth,
            confidence=fr.confidence,
            next_year_frequency=fr.next_year_frequency,
            engine_used="fallback",
        ))
    return result
```

### 3. Перенести import json на уровень модуля (skill_forecast.py)

Добавить `import json` в начало файла (строка 8), удалить из `fit()` (строка 67).

### 4. Исправить docstring (prophet_forecast.py:58-59)

**Было:**
```
falling back to GeneticForecastSkill
```

**Стало:**
```
falling back to SkillForecastEngine
```

### 5. Использовать months параметр (skill_forecast.py:152)

**Было:** `days_ahead = 365`

**Стало:** `days_ahead = months * 30`

### 6. Добавить валидацию months (оба модуля)

```python
def predict(self, skill: str, months: int = 12):
    if months < 1 or months > 60:
        return Err(DomainError(f"months must be 1-60, got {months}"))
```

### 7. Исправить engine_used (prophet_forecast.py:175)

Заменить `"genetic_fallback"` → `"fallback"` (прошлое название неактуально).
